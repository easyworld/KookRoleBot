# KookRoleBot

基于 [Kook.Net](https://github.com/gehongyan/Kook.Net) 的 KOOK 角色权限管理机器人，使用 .NET 10 和 SQLite。

## 功能

- 仅响应拥有 **管理员角色**（可在 `config.json` 中配置，默认为"管理员"）的用户的 @mention 消息
- 通过命令为指定用户授予角色并设置过期时间
- 支持过期时间累加（已有未过期记录时在原时间上追加）
- 每分钟自动扫描并移除过期角色
- 完善的错误提示与日志输出

## 命令格式

```
@机器人 @用户 角色名 +时长
```

> **注意：** 角色名与 `+时长` 之间必须有一个空格，例如 `vip +7d` 而非 `vip+7d`。

**示例：**

```
@机器人 @用户1 vip +7d
```

给用户1添加 vip 角色，有效期 7 天。

**支持多用户：**

```
@机器人 @用户1 @用户2 vip +3d
```

同时给多个用户添加相同角色和时长。

**时长格式：**

| 格式 | 含义 |
|------|------|
| `+1d` | 1 天 |
| `+7d` | 7 天 |
| `+30d` | 30 天 |

## 前置条件

- .NET 10 SDK
- KOOK Bot Token（在 [KOOK 开发者平台](https://developer.kookapp.cn/) 创建机器人获取）
- 机器人需要拥有 **管理角色** 权限

## 配置

首次运行时，程序会自动检测 `config.json` 是否存在。若不存在，将自动生成模板配置文件并提示填写：

```
已生成配置文件 config.json，请填写 Kook 机器人的 Bot Token 后重新启动。
```

打开 `config.json`，将 `Token` 替换为你的 Kook Bot Token：

```json
{
  "Token": "在此填写你的 Kook Bot Token",
  "DatabasePath": "kookbot.db",
  "AdminRoleName": "管理员"
}
```

| 字段 | 说明 |
|------|------|
| `Token` | Kook Bot Token |
| `DatabasePath` | SQLite 数据库文件路径，默认 `kookbot.db` |
| `AdminRoleName` | 允许操作的管理员角色名称，默认 `"管理员"` |

填写完成后重新启动即可。

## 运行

```bash
dotnet run
```

## 服务器设置

1. 将机器人邀请到 KOOK 服务器
2. 在服务器中创建管理员角色（名称需与 `config.json` 中 `AdminRoleName` 一致，默认为"管理员"）
3. 创建需要管理的业务角色（如 `vip`、`premium` 等）
4. 确保 bot 角色拥有 **管理角色** 权限，且 bot 角色高于被管理的角色

## 回复示例

成功授予权限时，机器人会回复：

```
✅ 用户1#1234 → vip 已授予，到期时间：2026-04-28 15:30:00
```

用户已有该角色并延期时：

```
✅ 用户1#1234 → vip 已延期，到期时间：2026-05-05 15:30:00
```

## 日志输出

机器人运行时会在控制台输出详细日志：

```
[Role]   授予角色 "vip" 给 用户1#1234
[Timer]  用户1#1234 的角色 "vip" 将于 2026-04-28 15:30:00 过期
[Reply]  ✅ 用户1#1234 → vip 已授予，到期时间：2026-04-28 15:30:00
[Expired] 已移除 用户1#1234 的过期角色 "vip"
```

## 错误处理

机器人会自动捕获常见 API 错误并给出中文解决提示：

| 错误 | 提示 |
|------|------|
| 403 权限不足 | 请给 Bot 角色开启「角色管理」权限，并确保 Bot 角色排在被管理的角色之上 |
| 404 未找到 | 目标用户或角色不存在，请检查角色名称 |
| 400 参数错误 | 请求参数错误，请检查命令格式 |
| 429 限流 | 操作过于频繁，请稍后再试 |

## 工作流程

```
管理员发送: @机器人 @用户A vip +7d
                    │
                    ▼
            校验发送者是否有管理员角色（AdminRoleName）
                    │
                    ▼
            解析目标用户、角色名、时长
                    │
                    ▼
            查找服务器中的角色并赋予用户
                    │
                    ▼
            记录/更新过期时间到 SQLite
                    │
                    ▼
            回复: ✅ 用户A → vip 已授予，到期时间：2026-04-28 15:30:00
```

**过期清理：** 机器人每分钟自动扫描数据库，移除已过期的角色并输出日志。
